# Contributing to Claude Code Agent Skills

First off, thank you for considering contributing! It's people like you that make this project great for everyone.

## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check the existing issues as you might find out that you don't need to create one. When you are creating a bug report, please include as many details as possible:

* **Use a clear and descriptive title**
* **Describe the exact steps to reproduce the problem**
* **Provide specific examples to demonstrate the steps**
* **Describe the behavior you observed and what behavior you expected**
* **Include screenshots if possible**

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion, please include:

* **Use a clear and descriptive title**
* **Provide a detailed description of the suggested enhancement**
* **List some examples of how this enhancement would be used**
* **Explain why this enhancement would be useful**

### Pull Requests

* Fill in the required template
* Follow the document style
* Include comments in your code where necessary
* Update documentation as needed

## Styleguides

### Git Commit Messages

* Use the present tense ("Add feature" not "Added feature")
* Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
* Limit the first line to 72 characters or less
* Reference issues and pull requests liberally after the first line

### Documentation Styleguide

* Use [Markdown](https://daringfireball.net/projects/markdown/)
* Keep code fences language-specific (e.g., \`\`\`typescript)
* Use bilingual format (English + 简体中文) for user-facing documentation
* Include practical examples wherever possible

## Additional Notes

Thank you for contributing to Claude Code Agent Skills! 🚀

# 贡献指南

首先，感谢你考虑为本项目做出贡献！正是因为有你这样的人，这个项目才能如此出色。

## 我如何贡献？

### 报告 Bug

在创建 Bug 报告之前，请检查现有问题，因为你可能发现不需要创建新的。创建 Bug 报告时，请包含尽可能多的细节：

* **使用清晰描述性的标题**
* **描述重现问题的确切步骤**
* **提供具体示例来演示这些步骤**
* **描述你观察到的行为和预期的行为**
* **如果可能，附上截图**

### 建议增强功能

增强建议通过 GitHub issue 追踪。创建增强建议时，请包含：

* **使用清晰描述性的标题**
* **详细描述建议的增强功能**
* **列出一些如何使用此增强功能的示例**
* **解释为什么这个增强功能会有用**

### 拉取请求

* 填写要求的模板
* 遵循文档风格
* 在必要时添加代码注释
* 根据需要更新文档

## 风格指南

### Git 提交信息

* 使用现在时态（"Add feature"而不是"Added feature"）
* 使用祈使语气（"Move cursor to..."而不是"Moves cursor to..."）
* 第一行限制在 72 个字符以内
* 在第一行之后自由引用问题和拉取请求

### 文档风格指南

* 使用 [Markdown](https://daringfireball.net/projects/markdown/)
* 保持代码块指定语言（例如 \`\`\`typescript）
* 面向用户的文档使用双语格式（English + 简体中文）
* 尽可能包含实用示例
